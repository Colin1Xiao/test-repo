## 💓 心跳检查（每次必做）

**收到心跳时执行：**

1. 运行健康检查脚本
2. 读取 `openclaw-health-check.json`
3. 检查状态变化 → 变化时报告，不变时 `HEARTBEAT_OK`

**状态变化报告格式：**
```
【小龙状态更新】

{🟢|🟡|🔴} Gateway：{变化}
{🟢|🟡|🔴} Telegram：{变化}
{🟢|🟡|🔴} Memory Search：{变化}
{🟢|🟡|🔴} 交易系统：{变化}

🧭 系统健康度：{🟢正常|🟡部分受限|🔴需要关注}
```

**告警分级：**
- 🔴 Critical：立即通知，不限时间
- 🟡 Warning：08:00-23:00 通知
- 🟢 Info：不通知，记录即可

## 🎯 状态驱动行为（核心规则）

**先看状态，再行动。**

| 状态 | 行为 |
|------|------|
| 🟢 正常 | 全功能正常，可执行高风险操作 |
| 🟡 警告 | 降级执行，避免依赖异常组件 |
| 🔴 严重 | 仅诊断和修复建议，停止外部操作 |

**静默保护机制：**

| 组件异常 | 限制 |
|---------|------|
| Gateway 🔴 | 禁止 Telegram 发送，禁止 workspace 外文件写入 |
| Memory Search 🔴 | 禁用记忆搜索，提示用户 |
| Telegram 🔴 | 仅被动回复，禁止主动发送 |
| 交易系统 🔴 | 禁止下单，仅监控和警告 |
| 全局 🔴 | 仅诊断修复，禁止外部操作 |

## 🔧 升级后自动检查

**每次完成工具升级后自动执行：**

```bash
~/.openclaw/workspace/scripts/post-upgrade-check.sh
```

### 检查项目

1. **brew 链接** - 检查并修复 Python 等 brew 包的链接
2. **PATH 配置** - 验证 `~/.zshrc` 和当前 PATH
3. **Python 环境** - 验证 python3/pip3 版本和可用性
4. **npm 环境** - 验证 npm 版本
5. **清理缓存** - brew cleanup + npm cache clean

### 自动修复

- Python 链接断开 → `brew link --overwrite python@3.14`
- PATH 缺失 → 添加到 `~/.zshrc`
- 缓存清理 → 自动执行

### 输出示例

```
=== 1. brew 链接 ===
  ✓ Python 3.14 已链接

=== 2. PATH 配置 ===
  ✓ ~/.zshrc PATH 已配置
  ✓ 当前 PATH 正确

=== 3. Python 环境 ===
  ✓ Python 3.14.3
  ✓ pip3 可用

=== 4. npm 环境 ===
  ✓ npm v11.12.1

=== 5. 清理缓存 ===
  ✓ brew 缓存已清理
  ✓ npm 缓存已清理

✓ 检查完成
```

## 🧠 Self-Improving 工作流

**学习日志目录：** `.learnings/`

### 触发条件

**记录到 `.learnings/LEARNINGS.md`：**
- 用户纠正 ("不对，应该是...")
- 知识缺口 (发现理解有误)
- 最佳实践 (发现更好方法)
- 工作流改进 (重复任务优化)
- 工具陷阱 (工具使用注意点)

**记录到 `.learnings/ERRORS.md`：**
- 命令失败 (非零退出码)
- 异常/崩溃
- 超时/连接问题
- 权限问题

**记录到 `.learnings/FEATURE_REQUESTS.md`：**
- 用户请求新功能
- 缺失的能力

### 晋升规则

**满足以下任一条件 → 晋升到 workspace 文件：**
- 同一模式出现 ≥3 次
- 跨 2+ 不同任务重复
- 30 天内重复发生

**晋升目标：**
| 学习类型 | 晋升到 |
|----------|--------|
| 行为模式 | `SOUL.md` |
| 工作流改进 | `AGENTS.md` |
| 工具陷阱 | `TOOLS.md` |
| 通用知识 | `MEMORY.md` |

### 快速命令

```bash
# 添加学习条目
~/.openclaw/workspace/scripts/add-learning.sh <lrn|err|feat> "标题" "类别"

# 查看待处理
grep -h "Status\*\*: pending" .learnings/*.md | wc -l

# 查找高优先级
grep -B5 "Priority\*\*: high" .learnings/*.md | grep "^## \["
```
