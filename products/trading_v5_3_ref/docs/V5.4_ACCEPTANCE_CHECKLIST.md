## 一、隐藏坑（必须避免）

### ⚠️ 坑1: Position Gate 误判 (OKX 常见)

```python
# ❌ 错误: 只读一个字段
position['contracts']

# ✅ 正确: 兼容多种字段
size = float(
    p.get("contracts") or
    p.get("positionAmt") or
    p.get("pos") or
    0
)
```

**后果**: 明明有仓位 → 系统认为没有 → 叠仓

---

### ⚠️ 坑2: 止损挂单失败 (最常见)

```python
# ❌ 危险: 不处理失败
exchange.create_order(...)

# ✅ 正确: try + fallback
try:
    exchange.create_order(...)
except Exception as e:
    trade["stop_loss_failed"] = True
```

**后果**: 你以为有止损，其实没有

---

### ⚠️ 坑3: Exit Source 误标 (逻辑错误)

```python
# ❌ 错误: 根据 PnL 判断
if pnl > 0:
    exit_source = "TAKE_PROFIT"

# ✅ 正确: 根据触发原因
if stop_loss_triggered:
    exit_source = "STOP_LOSS"
elif take_profit_triggered:
    exit_source = "TAKE_PROFIT"
```

**原则**: 来源 ≠ 结果

## 二、硬验收清单

### ✅ 1. Position Gate

**测试方法**: 连续发送 2 次相同信号

**预期结果**: 只开 1 笔

**失败标志**: 2 笔叠加

---

### ✅ 2. Stop Loss

**测试方法**: 入场后检查交易所订单列表

**预期结果**: 能看到止损单

**失败标志**: 无止损单

---

### ✅ 3. Trade 数据完整性

每笔交易必须:

| 字段 | 要求 |
|------|------|
| entry_price | ≠ 0 |
| exit_price | ≠ 0 |
| pnl | ≠ 固定值 |
| exit_source | 存在 |
| position_size | 存在 |

---

### ✅ 4. 不允许出现

| 现象 | 说明 |
|------|------|
| ❌ pnl = 全 0 | 数据链断裂 |
| ❌ exit = ticker (长期) | 未获取真实成交价 |
| ❌ 多笔叠仓 | Position Gate 失效 |
