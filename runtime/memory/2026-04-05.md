## ✅ GitHub Actions Test Gate 修复 — 完成

**时间**: 2026-04-05 14:00-14:21 CST

**问题**: GitHub Actions 测试失败 (5 个测试套件，多个测试失败)

**根因分析**:
1. 测试框架混用：`node:test` vs Jest
2. MockRedis 时间模拟不完整
3. 并发测试未共享 Redis 实例
4. TTL 单位混淆（秒 vs 毫秒）
5. Deployments 状态机定义错误

**修复内容**:

### 1. 测试框架迁移
- 从 `node:test` 迁移到 Jest (`@jest/globals`)
- 文件：`state_sequence.test.ts`, `recovery_coordinator.test.ts`
- 导入变更：`describe/it/beforeEach` → `@jest/globals`

### 2. MockRedisClient 时间模拟修复
```typescript
// 新增字段
private timeOffset = 0;  // 模拟时间偏移

// 统一时间获取
private now(): number {
  return Date.now() + this.timeOffset;
}

// 修复 advanceTime
advanceTime(ms: number): void {
  this.timeOffset += ms;
  // 清理过期 TTL
  for (const [key, ttl] of this.ttls.entries()) {
    if (this.now() > ttl) {
      this.store.delete(key);
      this.ttls.delete(key);
    }
  }
}
```

### 3. 并发测试共享 Redis 实例
```typescript
// 修复前：每个 coordinator 独立 Redis
const { coordinator: c1 } = createCoordinator('instance-1');
const { coordinator: c2 } = createCoordinator('instance-2');

// 修复后：共享同一个 Redis
const redis = new MockRedisClient();
const c1 = new RecoveryCoordinator(redis, audit, {}, 'instance-1');
const c2 = new RecoveryCoordinator(redis, audit, {}, 'instance-2');
```

### 4. TTL 单位修复
- Redis `ex` 选项是**秒**不是毫秒
- 测试等待时间从 150ms 改为 1500ms

### 5. Deployments 状态机修复
```typescript
// 修复前
terminal_states: ['completed', 'rolled_back']  // ❌ rolled_back 是终端状态

// 修复后
terminal_states: ['completed']  // ✅ rolled_back 可重试回 planned
```
