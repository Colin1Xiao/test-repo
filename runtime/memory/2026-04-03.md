## 🎯 核心结论

**Claude Code 真正价值**: 不是某个单独功能，而是把 `工具 + 权限 + 状态 + 任务 + 记忆 + 远程审批` 做成了统一 runtime。

**OpenClaw 升级目标**: 从命令驱动型助手 → 任务驱动、策略驱动、记忆驱动的 **Agent Runtime OS**

## 📋 12 个最高价值升级点（用户分析）

### 1️⃣ buildTool/buildSkill 工厂（第一优先级）

**Claude Code 实现**: `src/Tool.ts` - `buildTool()` 工厂 + `ToolUseContext`

**解决的问题**:
- 所有工具共享相同生命周期
- 所有工具都能挂权限逻辑
- 所有工具都能挂流式 UI
- 所有工具都能挂日志和 telemetry
- 所有工具都能挂中断与恢复
- 所有工具都能被统一搜索、发现、调度

**OpenClaw 行动**:
```typescript
buildSkill/buildTool/buildAction 统一工厂
元数据：name, description, inputSchema, outputSchema, searchHint, 
        readOnly, destructive, concurrencySafe, supportsStreaming, 
        shouldDefer, requiresApproval, call(ctx, input)
```

**依赖关系**: 后面所有升级都依赖它（权限/审批/hooks/任务/MCP/插件/UI/远程桥接）

### 2️⃣ ExecutionContext 统一执行上下文

**OpenClaw 行动**:
```typescript
ExecutionContext 最少包括:
- session id, task id, agent id
- workspace root, abort signal
- logger, event emitter
- memory facade, permission evaluator
- ui/telegram bridge, fs facade, exec facade
```

**价值**: 每个 skill 天然拥有取消/进度/审批/记忆/任务/UI/远程会话同步能力

### 3️⃣ 权限系统从"开关"升级成"规则引擎"

- 多来源规则合并：system default / agent policy / workspace policy / local config / session override / user one-shot
- 决策原因追踪（人类可解释）
- 精确匹配：exact / prefix / wildcard / tool scoped / mcp server scoped

**OpenClaw 行动**: 升级为规则引擎，支持多来源/多级结果/原因追踪/精确匹配

**直接收益**: 从"危险但好用" → "强但可控"

### 5️⃣ 查询状态机 + 任务状态机（行为稳定性）

**Claude Code 实现**: 
- `src/utils/QueryGuard.ts` - Turn 状态机 (idle/dispatching/running)
- `src/Task.ts` - Task 状态机 (pending/running/completed/failed/killed)

**解决的问题**:
- 异步空档重入
- Telegram 多消息并发
- bot 重复触发
- cancel 后 stale finally 清理
- 中断后状态丢失
- 无法恢复/无法定位当前步骤

**OpenClaw 行动**: 两层状态机
```typescript
// A. Turn/Query 状态机
idle | dispatching | running

// B. Background Task 状态机
created | queued | running | waiting_approval | waiting_input | completed | failed | cancelled
```

**直接价值**: 行为稳定性 —— 解决"蠢 agent"本质问题（状态混乱/重入/丢失）

## 🔍 我的分析（小龙视角）

### 架构风险评估

**当前 OpenClaw 状态**:
- ✅ Skills 系统已存在，但缺少统一工厂抽象
- ✅ 权限系统有基础（allowlist/deny），但缺少规则引擎
- ✅ 状态管理分散（health-check/recovery-history 等），缺少单一真相源
- ✅ 记忆系统有（MEMORY.md + daily files），但缺少程序化读写接口
- ⚠️ 任务系统薄弱（cron 有基础，但缺少 Task 抽象）
- ⚠️ 执行上下文不统一（各 skill 自己处理 session/workspace）

### 关键依赖链

```
buildTool 工厂 (P0)
    ↓
ExecutionContext (P0)
    ↓
权限规则引擎 (P1)
    ↓
状态Store 拆分 (P1)
    ↓
任务系统 (P2)
    ↓
记忆程序化接口 (P2)
    ↓
远程审批桥接 (P3)
```

### 风险识别

1. **过度工程化风险**: Claude Code 是桌面应用，OpenClaw 是消息驱动 —— 不能照搬
2. **性能风险**: 统一状态树可能成为瓶颈（需拆分 + 事件驱动）
3. **兼容性风险**: 现有 36+ skills 需要迁移适配
4. **复杂度风险**: 权限规则引擎可能变得难以调试

## 🐉 OpenClaw 网关 SIGTERM 重启问题（2026-04-03 17:24）

### 问题现象
- 网关频繁重启（17:00-17:20 期间重启 5+ 次）
- 每次执行 `openclaw plugins list` 或 `openclaw gateway status` 后重启
- WebChat 连接断开（code=1012 reason=service restart）

### 根本原因
1. **OpenClaw CLI 行为**：执行 `openclaw gateway *` 命令时发送 SIGTERM 信号给网关进程
2. **LaunchAgent 配置**：`KeepAlive: true` 导致收到 SIGTERM 后立即自动重启
3. **两者结合**：CLI 查询 → SIGTERM → 自动重启 → WebChat 断开

### 日志证据
```
16:49:48 - SIGTERM received (plugins list 命令)
16:55:00 - SIGTERM received (plugins list 命令)
16:58:40 - SIGTERM received (plugins list 命令)
17:00:18 - SIGTERM received (plugins list 命令)
17:07:20 - SIGTERM received (plugins list 命令)
17:08:54 - SIGTERM received (plugins list 命令)
17:11:00 - SIGTERM received (plugins list 命令)
17:13:27 - SIGTERM received (plugins list 命令) → WebChat 断开
17:16:43 - SIGTERM received (plugins list 命令) → WebChat 断开
17:20:21 - 网关重启后稳定运行
```

### 修复方案

**修改 LaunchAgent 配置** (`~/Library/LaunchAgents/ai.openclaw.gateway.plist`)：

```xml
<!-- 修复前 -->
<key>KeepAlive</key>
<true/>

<!-- 修复后 -->
<key>KeepAlive</key>
<dict>
  <key>SuccessfulExit</key>
  <false/>
  <key>Crashed</key>
  <true/>
</dict>
<key>ExitTimeOut</key>
<integer>30</integer>
```

**效果**：
- 仅在崩溃时自动重启（非正常退出不重启）
- CLI 命令不再触发重启循环
- 优雅关闭时间 30 秒

### 安全监控方法（不触发重启）

```bash
# ✅ 安全 - 直接读日志
tail -20 /tmp/openclaw/openclaw-2026-04-03.log

# ✅ 安全 - 直接读进程
ps aux | grep openclaw-gateway | grep -v grep

# ✅ 安全 - 直接读 plist
cat ~/Library/LaunchAgents/ai.openclaw.gateway.plist

# ✅ 安全 - HTTP API 查询
curl http://127.0.0.1:18789/health

# ❌ 禁止 - 会触发重启
openclaw gateway status
openclaw plugins list
openclaw gateway restart
```

### 当前状态（17:24）
- 网关进程：正常运行 (PID 11859)
- 最后启动：17:20:21（已稳定运行 4 分钟）
- SIGTERM：无（修复后）
- WebChat：需刷新浏览器重新连接

### 经验教训
1. CLI 命令可能触发副作用（SIGTERM）
2. LaunchAgent 配置需要仔细审查 KeepAlive 行为
3. 监控工具不应该影响被监控系统
4. 优先使用只读方法（日志文件、HTTP API）而非 CLI 命令
