## 四、后续推进顺序

### Phase 1: 24小时首轮观察（当前）

**重点盯**: 
- [ ] 是否异常退出
- [ ] 是否自动重启
- [ ] 是否出现重复挂单/重复告警
- [ ] 是否有订单状态与持仓状态不一致
- [ ] 是否有挂单超时后未正确撤单/重挂
- [ ] 是否有紧急平仓后未进入冷静期

**记录表**: `DAILY_REVIEW_TEMPLATE.md`

---

### Phase 2: 运行中故障演练

**必做6个场景**:

1. **OKX测试网连接中断**
   - 检测连接中断
   - 停止新订单
   - 记录错误
   - 发送告警
   - 进入恢复状态

2. **API超时/429/500**
   - 捕获API错误
   - 停止风险动作
   - 指数退避重试
   - 持续失败降级

3. **Telegram失败**
   - 不阻塞交易
   - 落本地critical日志
   - 恢复后补发

4. **压低日限额触发风控**
   - 到达限额停止开仓
   - 允许减仓/平仓
   - 状态标记清晰

5. **注入回撤保护**
   - 触发冷静期
   - 禁止新增高风险
   - 允许减仓/平仓

6. **模拟部分成交**
   - 持仓按实际更新
   - 平仓绑定实际仓位
   - 无双倍仓位误判

**记录表**: `FAULT_DRILL_RECORD.md`

---
