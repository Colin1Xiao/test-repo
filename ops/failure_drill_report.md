## 演练目标

验证系统在异常情况下的行为是否符合预期：

1. **状态一致性** → 异常后状态不丢失、不混乱
2. **恢复能力** → 可从中断点恢复，不重复执行
3. **安全边界** → 危险操作被正确拦截
4. **可观测性** → 所有异常有日志、可追溯

## 演练 A: 审批卡住

### 场景描述

模拟任务进入 `waiting_approval` 状态后，用户长期不响应。

### 测试步骤

```bash
# 1. 触发需要审批的任务
# (执行一个需要审批的命令，例如 workspace 外写入)

# 2. 确认任务状态
cat ~/.openclaw/runtime/task_store.json | jq '.[] | select(.status == "waiting_approval")'

# 3. 不响应审批，等待 5 分钟

# 4. 检查任务状态是否保持
# (应该仍然是 waiting_approval，没有超时或丢失)

# 5. 检查 ApprovalStore 一致性
cat ~/.openclaw/runtime/approval_store.json | jq

# 6. 手动恢复任务
# (通过 Telegram 或 WebChat 批准/拒绝)

# 7. 验证任务继续执行或终止
# (不应重复执行，不应卡死)
```

|--------|---------|---------|
| 任务状态保持 | waiting_approval 不变 | ⏳ |
| ApprovalStore 一致 | 状态与 task_store 一致 | ⏳ |
| 无重复执行 | 批准后只执行一次 | ⏳ |
| 可恢复 | 批准后可继续执行 | ⏳ |
| 日志完整 | 有完整的审批等待日志 | ⏳ |

## 演练 C: Worktree 中断

### 场景描述

模拟在隔离 worktree 中执行任务时中断 (进程崩溃/手动停止)。

### 测试步骤

```bash
# 1. 创建 worktree
# (触发一个需要 worktree 的高风险任务)

# 2. 记录 worktree 路径
WORKTREE_PATH=$(ls -td ~/.openclaw/workspace/worktrees/* | head -1)
echo "Worktree: $WORKTREE_PATH"

# 3. 在任务执行中中断
# (Ctrl+C 或 kill 进程)

# 4. 检查 worktree 状态
ls -la $WORKTREE_PATH

# 5. 检查是否产出 diff.patch
cat $WORKTREE_PATH/diff.patch 2>/dev/null || echo "No diff.patch"

# 6. 检查 cleanup 是否回收
ls -la ~/.openclaw/workspace/worktrees/

# 7. 验证主工作区未受影响
# (检查主目录文件是否未被修改)
```

### 预期行为
