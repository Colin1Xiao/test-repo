## 四、当前风控基线

建议当前默认参数如下：

```json
{
  "daily_limit": 500,
  "weekly_limit": 2000,
  "monthly_limit": 8000,
  "max_drawdown": 0.10
}
```

### 风控含义

- 日亏损上限：`500 USD`
- 周亏损上限：`2000 USD`
- 月亏损上限：`8000 USD`
- 最大回撤阈值：`10%`
- 触发回撤保护后进入冷静期，暂停高风险动作

## 七、下一步建议

当前阶段不建议继续扩展功能，建议转入 **联调验证 + 灰度验证**。

### 1. 全链路联调

重点验证：

- 数据采集 → 信号生成 → 风控判断 → 下单执行 → 告警通知
- 各模块输入输出是否一致
- 配置更新是否正确生效

### 2. 故障演练

建议人工模拟以下场景：

- OKX 不可用
- Binance 延迟升高
- API 返回异常
- Telegram 通知失败
- 达到日 / 周 / 月风险上限
- 触发 10% 回撤保护

### 3. 模拟盘 / 小资金灰度

建议先进行低风险验证，不直接进入高风险实盘。

### 4. 回测与实盘一致性核查

重点核对：

- 手续费
- 滑点
- 成交延迟
- 部分成交
- 仓位更新逻辑
- 止损执行逻辑

### 5. 上线检查清单

建议补齐正式上线前 checklist，包括：

- API 权限检查
- 风控参数检查
- 主备切换检查
- 告警链路检查
- 回滚方案检查
