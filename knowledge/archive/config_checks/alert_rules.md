## 告警通知模板

### WARNING 模板

```
🟡 OpenClaw 告警 [WARNING]

规则: {rule_name}
时间: {timestamp}
详情: {alert_details}
建议: {suggestion}

当前状态: 需关注，请观察趋势
```

### CRITICAL 模板

```
🔴 OpenClaw 告警 [CRITICAL]

规则: {rule_name}
时间: {timestamp}
详情: {alert_details}
影响: {impact_description}
建议: {suggestion}

当前状态: 需立即处理
```

## 告警处理流程

```
告警触发
    ↓
检测去重（5-60 分钟窗口）
    ↓
发送通知（Telegram + 日志）
    ↓
人工介入（CRITICAL 5 分钟内）
    ↓
问题定位
    ↓
处理/修复
    ↓
验证恢复
    ↓
记录 incident
```

## 告警屏蔽规则

以下情况可临时屏蔽告警：

- [ ] 已知 provider 维护窗口
- [ ] 主动压测期间
- [ ] 已确认的非系统问题

**屏蔽流程：**
1. 记录屏蔽原因和时间
2. 设置自动恢复时间
3. 通知相关人员

## 规则 review 周期

- **每周：** 检查告警触发频率，调整阈值
- **每月：** review 规则有效性，增删规则
- **每季度：** 全面评估告警体系
