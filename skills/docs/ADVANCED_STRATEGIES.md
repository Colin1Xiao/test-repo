## 1. 黑天鹅防护策略 🛡️

**重要性**: ⭐⭐⭐⭐⭐ 必配策略

### 功能
实时检测极端行情并自动平仓避险：
- 闪崩检测（5 分钟跌幅 > 5%）
- 波动率暴增（ATR > 3 倍均值）
- 成交量异常（> 5 倍均值）
- 大趋势检测（15 分钟波动 > 8%）

### 使用方式

```bash
# 单独运行检测
python3 scripts/strategy_blackswan.py \
  --symbol BTC/USDT \
  --timeframe 5m

# 持续监控（推荐）
watch -n 30 'python3 scripts/strategy_blackswan.py --symbol BTC/USDT --json'
```

### 警报级别

| 级别 | 条件 | 动作 |
|------|------|------|
| 🟢 GREEN | 无异常 | 正常交易 |
| 🟡 YELLOW | 成交量放大 3-5x | 减仓 20-30%，收紧止损 |
| 🟠 ORANGE | 5 分钟跌 3-5% 或 ATR 2-3x | 减仓 50% 或清仓 |
| 🔴 RED | 5 分钟跌 >5% 或 ATR >3x | **立即清仓** |

### 集成到监控系统

```python
# 在 monitor.py 中添加黑天鹅检测
from strategy_blackswan import detect_black_swan

# 每次生成信号前检查
alerts = detect_black_swan(df)
if alerts['should_close']:
    print(f"🚨 黑天鹅警报！{alerts['reasons']}")
    # 执行平仓
    close_all_positions()
```

### 实战案例

**2024-08-05 BTC 闪崩**:
- 5 分钟从 68,000 → 62,000 (-8.8%)
- 黑天鹅策略在 -5% 时触发 RED 警报
- 自动清仓避险，避免后续 -15% 亏损

## 2. 金字塔滚仓策略 📐

**适用场景**: 确认的单边大趋势

### 原理
在盈利情况下分批加仓，扩大收益：

```
初始仓位：40%  @ 价格 100
  ↓ 盈利 2%
第 1 次加仓：30% @ 价格 102
  ↓ 盈利 4%
第 2 次加仓：20% @ 价格 104
  ↓ 盈利 6%
第 3 次加仓：10% @ 价格 106
```

### 使用方式

```bash
# 查看加仓计划
python3 scripts/strategy_pyramid.py \
  --symbol BTC/USDT \
  --capital 10000 \
  --side long \
  --entry 68500

# 持续监控加仓机会
python3 scripts/strategy_pyramid.py \
  --symbol BTC/USDT \
  --entry 68500 \
  --side long \
  --level 0 \
  --timeframe 5m
```

### 参数配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `capital` | 10000 | 总资金 (USDT) |
| `entry` | 必需 | 初始入场价格 |
| `side` | long | 方向 (long/short) |
| `level` | 0 | 当前加仓级别 (0-3) |

### 加仓规则

| 级别 | 仓位比例 | 加仓条件 | 止损位置 |
|------|----------|----------|----------|
| 0 (初始) | 40% | 趋势确认 | -2% |
| 1 | 30% | 盈利 +2% | +1% (锁定利润) |
| 2 | 20% | 盈利 +4% | +2% |
| 3 | 10% | 盈利 +6% | +3% |

### 关键要点

✅ **必须遵守**:
1. 只在强趋势中使用（趋势强度 > 3）
2. 每次加仓后必须上移止损
3. 趋势反转立即清仓（不要犹豫）
4. 总仓位 ≤ 50% 资金

❌ **禁止使用**:
- 震荡行情
- 趋势不明朗
- 黑天鹅警报期间

### 实战案例

**BTC 2024 牛市趋势**:
- 入场：60,000 (40% 仓位)
- +2% → 61,200 加仓 30%，止损上移至 60,500
- +4% → 62,400 加仓 20%，止损上移至 61,200
- +6% → 63,600 加仓 10%，止损上移至 62,000
- 最终趋势结束于 72,000，总收益 +28%（vs 单利 +20%）

## 3. 专业止损管理 🛑

**重要性**: ⭐⭐⭐⭐⭐ 生存第一

### 止损方法对比

#### 1. 固定百分比止损

```bash
python3 scripts/stoploss_manager.py \
  --symbol BTC/USDT \
  --entry 68500 \
  --side long \
  --method fixed \
  --stop-pct 2.0
```

**适用**: 所有场景  
**优点**: 简单明确  
**缺点**: 不考虑波动率

---

#### 2. ATR 动态止损

```bash
python3 scripts/stoploss_manager.py \
  --symbol BTC/USDT \
  --entry 68500 \
  --side long \
  --method atr \
  --atr-multiplier 2.5
```

**原理**: 止损 = 入场价 ± (ATR × 倍数)

| 波动率 | ATR | 推荐倍数 | 止损幅度 |
|--------|-----|----------|----------|
| 低 | <1% | 2.0x | ~2% |
| 中 | 1-2% | 2.5x | ~3-5% |
| 高 | >2% | 3.0x | ~6%+ |

**适用**: 不同波动率环境  
**优点**: 自适应市场  
**优点**: 避免被正常波动震出

---

#### 3. 支撑/阻力位止损

```bash
python3 scripts/stoploss_manager.py \
  --symbol BTC/USDT \
  --entry 68500 \
  --side long \
  --method support_resistance
```

**原理**: 
- 做多：止损设在支撑位下方 0.5%
- 做空：止损设在阻力位上方 0.5%

**适用**: 有明确支撑阻力的行情  
**优点**: 符合技术分析  
**缺点**: 支撑阻力可能被假突破

---

#### 4. 追踪止损（移动止损）

```bash
python3 scripts/stoploss_manager.py \
  --symbol BTC/USDT \
  --entry 68500 \
  --side long \
  --method trailing \
  --trailing-pct 2.0 \
  --current 70000
```

**原理**: 止损价随盈利上移（只上不下）

```
入场：68,500，初始止损：67,130 (-2%)
价格上涨至 70,000，止损上移至 68,600 (+1%)
价格上涨至 72,000，止损上移至 70,560 (+3%)
价格回调至 70,500，触发止损，盈利 +2.9%
```

**适用**: 趋势行情锁定利润  
**优点**: 让利润奔跑  
**优点**: 保护既有盈利

---

#### 5. 分级止损（部分平仓）

```bash
python3 scripts/stoploss_manager.py \
  --symbol BTC/USDT \
  --entry 68500 \
  --side long \
  --method tiered
```

**原理**: 分多级止损，每级平掉部分仓位

| 级别 | 亏损幅度 | 平仓比例 | 剩余仓位 |
|------|----------|----------|----------|
| 1 | -1% | 30% | 70% |
| 2 | -2% | 50% | 20% |
| 3 | -3% | 20% | 0% |

**适用**: 不确定是否反转时  
**优点**: 避免一次性止损在最低点  
**优点**: 保留翻盘机会

---

### 止损推荐配置

#### 日内交易（5-10x 杠杆）
```
方法：ATR 止损 (2.5x)
幅度：2-3%
止盈：4-6%
盈亏比：2:1
```

#### 波段交易（10-20x 杠杆）
```
方法：支撑阻力位 + 追踪止损
初始：支撑位下方 0.5%
移动：盈利 2% 后启用追踪 (2%)
```

#### 高杠杆交易（50x+）⚠️
```
方法：固定止损 + 分级止损
固定：1.5% 硬止损
分级：-1% 平 50%, -1.5% 全平
止盈：3%
```
