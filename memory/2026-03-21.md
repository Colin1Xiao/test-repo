# 📅 2026-03-21 小龙交易系统重大修复

## 🚨 关键问题诊断与修复

### 问题1：执行引擎阻塞导致系统停止

**现象**：
- 执行超时后系统卡死
- `Event loop is closed` 错误
- 队列积压，所有后续信号被跳过

**根因**：
- 每次执行创建新 event loop，超时后关闭
- 关闭的 loop 导致后续异步操作失败

**修复**：
- Execution Engine V2：固定 event loop，不关闭
- timeout 后正确 cancel task
- 文件：`core/execution_engine.py`

---

### 问题2：Strategy Guardian 误判

**现象**：
- 胜率 0% 触发 STOP
- 系统无法运行

**根因**：
- 使用 win_rate < 40% 作为 STOP 规则
- 尾部策略天然低胜率高赔率，win_rate 是"毒指标"

**修复**：
- Strategy Guardian V2：Edge 驱动模式
- 改为 Expectancy <= 0 → STOP
- 移除 win_rate STOP 规则
- 文件：`core/strategy_guardian.py`

---

### 问题3：数据链断裂 - PnL/Slippage 全为 0

**现象**：
```
avg_pnl: 0.0000%
avg_slippage: 0.0000%
signal_quality: 0.00
```

**根因**：
- 执行链把"成交确认"当"执行完成"
- 等待 fill 导致超时，返回 None
- None 被当作成功处理

**核心认知**：
> 下单成功 ≠ 成交确认
> 执行 = 非阻塞，确认 = 异步

**修复**：
- 创建 `minimal_executor.py` - 最小正确执行链
- 不等待 fill，下单成功立即返回 trade
- 降级机制：orderbook 失败用 ticker 价格

```

下单成功 = 执行成功

```
尾部策略 ≠ 高胜率策略
