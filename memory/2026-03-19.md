### 结构分类

| 结构 | 条件 | 交易模式 |
|------|------|----------|
| RANGE | 低波动 + 无趋势 | 均值回归 (阈值80) |

| CHAOTIC | 高波动 + 低流动性 | **禁止交易** |

### 结构优先级

```
CHAOTIC > BREAKOUT > TREND > RANGE
```

### 保护规则

- **CHAOTIC = 禁止交易**
- **结构跳变保护**: 结构变化后跳过 2 个周期

### 执行顺序

```
Signal → Structure Engine → AI Risk → Adaptive Leverage → Execution
```

## Regime Memory (V9) 完成 (19:32)

### 核心能力

| 模块 | 文件 | 功能 |
|------|------|------|
| 市场记忆 | `core/regime_memory.py` | 记住"自己在哪些市场死过" |

### 核心功能

1. **记录系统** - 每笔交易都记录环境 + 结果
2. **环境聚类** - 把市场环境离散化
3. **统计表现** - 胜率、平均收益
4. **决策支持** - 是否应该交易

### 环境桶 (Bucket)

```
(structure, volatility, volume, risk_level)
```

离散化：
- 波动率: LOW / MEDIUM / HIGH
- 成交量: LOW / NORMAL / HIGH

### 决策规则

| 条件 | 结果 |
|------|------|
| 样本 < 10 | 允许（数据不足） |
| 胜率 < 40% | 禁止 |
| 平均收益 < 0 | 禁止 |
| 否则 | 允许 |

### API 端点

| 端点 | 功能 |
|------|------|
| `/regime-memory` | 获取记忆状态 |
| `/regime-memory/check` | 检查是否应交易 |
| `/regime-memory/buckets` | 所有环境桶 |
| `/regime-memory/worst` | 最差环境 |
| `/regime-memory/best` | 最好环境 |
| `/regime-memory/dashboard` | Dashboard UI |

## 🚨 P0 事故修复 (21:45-21:55)

### 问题描述

**系统能开仓但不会平仓** - 核心风控失效

**现象**:
- 持仓累积到 4.84 ETH
- 持仓时间超过 300 秒
- 无自动平仓逻辑

**根因**:
1. `executor.open_positions` 是内存变量，重启后丢失
2. 系统缺少独立的 Position Lifecycle Engine
3. `close_position()` 只检查内存持仓，不查询交易所

### 紧急处置

1. **手动平仓**: 累计 4.84 ETH，亏损 -1.49%
2. **停止系统**: `pkill -f run_v52_live.py`
3. **创建模块**: `core/position_lifecycle.py` - Position Lifecycle Engine

### 新增模块

**PositionLifecycleEngine** (`core/position_lifecycle.py`):
- 止盈止损
- 时间退出 (30秒)
- 爆仓保护 (-0.4%)
- 强制退出机制

### 修复内容

1. **scoring_engine_v43.py**:
   - 添加 `get_threshold()` 方法
   - 动态阈值: RANGE=40, TREND=65, BREAKOUT=60
   - 修复 `stop_loss` 单位 bug

2. **live_executor.py**:
   - 添加 `import time`
   - `entry_time` 使用时间戳
   - `close_position()` 增加交易所持仓查询

3. **run_v52_live.py**:
   - 集成 `PositionLifecycleEngine`
   - 每周期强制检查退出条件

### 核心认知

> 交易系统不是因为"进不去"亏钱
> 而是因为"出不来"爆炸

**优先级**: EXIT > ENTRY
