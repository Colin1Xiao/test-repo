## 十一、关键数据验证

### Alerts 级别语义复核

**问题**: 历史页面显示 CRITICAL=0，但数据库有 409 条

**原因**: 
1. ✅ 数据正确：数据库 CRITICAL=409（最近 7 天）
2. ✅ API 正确：`/api/history/alerts` 返回 CRITICAL=409
3. ✅ 页面逻辑正确：使用 `summary` 字段统计
4. ⚠️ 显示问题：页面按 `ts DESC` 排序，最近 20 条是 INFO 恢复事件

**结论**: **数据语义正确，页面显示正常（需等待 JS 加载）**

### 历史 WARN 占比高的原因

**现象**: WARN=11075 (96.3%), CRITICAL=409 (3.6%), INFO=8 (0.1%)

**原因**:
- 旧格式告警回填时默认映射为 WARN
- 旧格式示例：`{"timestamp": "...", "alerts": ["执行质量过低"]}` → WARN
- 新格式告警（P1-2 后）有明确级别

**建议**: 
- 保持现状（历史数据保留原语义）
- 或添加 `source_mode: 'legacy' | 'dual_write'` 字段区分
