## 四、双执行防护

### 风险场景
```
信号触发 → ExecutionEngine.submit()
         → LiveExecutor.execute_signal()
         → 两次下单！
```

### 解决方案

在 `run_v52_live.py` 中只保留一个入口:

```python
# ❌ 禁用 ExecutionEngine
if self.execution_engine:
    self.execution_engine.stop()
    self.execution_engine = None

# ✅ 只用 LiveExecutor
result = await self.executor.execute_signal(...)
```

## 五、本地状态管理

在主系统添加:
```python
class TradingSystem:
    def __init__(self):
        self.current_position = None  # 本地持仓状态

    async def _execute_trade(self, ...):
        # 双重检查
        if self.current_position is not None:
            print("⛔ 本地状态: 已有持仓")
            return None

        result = await self.executor.execute_signal(...)

        if result:
            self.current_position = result
```
