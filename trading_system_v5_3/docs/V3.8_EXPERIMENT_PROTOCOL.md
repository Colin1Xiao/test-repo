# V3.8 Edge Validation 实验协议

## 📋 固定参数（30笔基线实验期间禁止修改）

| 参数 | 值 | 说明 |
|------|-----|------|
| `score_threshold` | 70 | 评分引擎入场阈值 |
| `stop_loss` | 0.5% | 硬止损比例 |
| `take_profit` | 0.2% | 止盈比例 |
| `time_exit` | 30s | 超时退出时间 |
| `base_risk_fraction` | 2% | 基础风险比例 |
| `capital_state` | 自动切换 | NORMAL/REDUCED/MICRO 按现有逻辑 |

## 📊 观察指标（30笔基线）

### 核心指标
- `trade_count`: 总交易笔数
- `win_rate`: 胜率 (%)
- `avg_win`: 平均盈利 (USDT)
- `avg_loss`: 平均亏损 (USDT)  
- `max_win`: 最大盈利 (USDT)
- `max_loss`: 最大亏损 (USDT)
- `profit_factor`: 盈亏比 = 总盈利 / 总亏损
- `expectancy`: 期望值 = (胜率 × avg_win) + (败率 × avg_loss)

### 分布指标
- `exit_distribution`: 退出原因分布 (STOP_LOSS/TAKE_PROFIT/TIME_EXIT)
- `capital_state_distribution`: 资金状态分布 (NORMAL/REDUCED/MICRO)

## 🧪 实验阶段

### 阶段1: 基线验证（30笔）
- **目标**: 收集30笔完整交易数据
- **约束**: 不允许任何参数调整
- **输出**: 上述所有指标的完整报告

### 阶段2: 单变量实验（仅当基线完成）
- **约束**: 一次只修改一个参数
- **选项A**: `score_threshold: 70 → 68`
- **选项B**: `regime detection 更严格`  
- **选项C**: `trailing/take profit 微调`

## ✅ 判定标准

### 继续实验
- `profit_factor > 1`
- `expectancy > 0`

### 可疑结果  
- 样本 < 30
- 指标波动过大

### 停止实验
- `profit_factor < 1` 且 `expectancy < 0` 持续

## 📝 实验记录

- 开始时间/结束时间
- 完整的交易日志
- 所有观察指标
- 任何异常情况
