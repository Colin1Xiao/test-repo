## 三、观测方法

### 3.1 自动化观测

**脚本**: `scripts/wave2-observe.sh`

```bash
#!/bin/bash
# Wave 2 观测脚本

# 1. 检查服务健康
curl -s http://localhost:3000/health | jq .

# 2. 检查文件大小
du -sh ~/.openclaw/workspace/openclaw-runtime/data/*/

# 3. 检查锁残留
ls -la ~/.openclaw/workspace/openclaw-runtime/data/locks/

# 4. 统计 JSONL 行数
wc -l ~/.openclaw/workspace/openclaw-runtime/data/*/incidents.jsonl
wc -l ~/.openclaw/workspace/openclaw-runtime/data/*/timeline.jsonl
wc -l ~/.openclaw/workspace/openclaw-runtime/data/*/audit.jsonl

# 5. 检查 P0 告警
# (通过告警系统)
```

**执行频率**: 每小时

### 3.2 手动观测

**检查清单**:

- [ ] 服务响应正常
- [ ] 文件无损坏
- [ ] 锁无残留
- [ ] 日志无异常
- [ ] P0 告警无触发

**执行频率**: 每 6 小时

### 3.3 重启验证

**步骤**:

1. 记录当前状态（incident 数/timeline 数）
2. 重启服务
3. 验证恢复数量一致
4. 验证查询功能正常
5. 记录结果

**执行频率**: 每 24 小时

## 五、报告机制

### 5.1 定期报告

| 报告 | 时间 | 内容 |
|------|------|------|
| 12h 状态报告 | T+12h | Phase 1 总结 |
| 24h 状态报告 | T+24h | Phase 1-2 总结 + 重启验证 |
| 48h 完成报告 | T+48h | Wave 2-A 总结 |

### 5.2 异常报告

**触发条件**:
- 任何 P0 告警触发
- 观测指标超阈值
- 发现数据不一致

**报告内容**:
- 异常描述
- 影响范围
- 已采取措施
- 建议行动
