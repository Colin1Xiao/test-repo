### 2. Alert Action API ✅

**文件**: `src/alerting/alert_actions.ts` (9.9KB)

**功能**:
- `acknowledge(alert)` — 确认告警
- `silence(alert, duration)` — 静音告警
- `escalate(alert)` — 升级告警
- `link_incident(alert, incident)` — 关联 incident
- `open_runbook(alert)` — 打开 runbook

**记录**:
- 所有动作写入 action_history
- 支持查询历史

### 3. Timeline Integration ✅

**文件**: `src/alerting/timeline_integration.ts` (8.9KB)

**功能**:
- 告警动作写入 Timeline
- 支持按 alert/incident/correlation_id 查询
- 支持时间范围查询
- 支持标签过滤

**事件类型**:
- `alert_triggered`
- `alert_routed`
- `alert_acknowledged`
- `alert_silenced`
- `alert_escalated`
- `incident_linked`
- `incident_created`
- `incident_updated`
- `runbook_opened`
- `runbook_action`
- `runbook_completed`
- `recovery_action`

**查询示例**:
```typescript
const store = getTimelineStore();

// 按告警查询
const alertEvents = store.getByAlert('RedisDisconnected', 100);

// 按 incident 查询
const incidentEvents = store.getByIncident('incident-xxx', 100);

// 按 correlation 查询
const correlationEvents = store.getByCorrelation('corr-xxx', 100);

// 最近事件
const recent = store.getRecent(100);
```

### 4. Incident Repository ✅

**文件**: `src/alerting/incident_repository.ts` (11.9KB)

**功能**:
- Incident 持久化
- 防重复创建（相同 correlation/resource/alert）
- 自动关联已有 incident
- 状态管理 (open/investigating/resolved/closed)

**防重复逻辑**:
1. 检查相同 correlation_id 的 open incident
2. 检查相同 resource 的 open incident（同类型）
3. 检查相同 alert 的 open incident（5 分钟内）

**查询示例**:
```typescript
const repo = getIncidentRepository();

// 创建或获取 incident
const { incident, created } = repo.createOrGet({
  type: 'redis_outage',
  severity: 'P0',
  title: 'Redis 连接断开',
  correlation_id: 'redis-001',
  resource: 'redis:primary',
  alert_name: 'RedisDisconnected',
  created_by: 'on-call',
});

// 查询 open incidents
const open = repo.getOpen(100);

// 按 correlation 查询
const byCorr = repo.getByCorrelation('redis-001', 100);

// 统计
const stats = repo.getStats();
```
