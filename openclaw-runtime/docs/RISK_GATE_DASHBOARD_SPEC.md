## 一、概述

### 目的

提供**风险门控仪表板**，实时监控 Gray 部署各阶段的健康状态和 Gate 条件满足度。

### 核心功能

| 功能 | 说明 |
|------|------|
| **Gate 状态可视化** | 显示当前 Gate (1/2/3) 及各阶段进度 |
| **条件满足度** | 实时显示硬性/软性条件满足情况 |
| **风险预警** | 接近阈值时自动预警 |
| **决策辅助** | 为 Gate 决策提供数据支持 |

### 设计原则

| 原则 | 说明 |
|------|------|
| **只读优先** | Gray 期间仅展示，不执行写操作 |
| **实时性** | 数据延迟 < 30 秒 |
| **直观性** | 一眼看清 Gate 状态 |
| **可追溯** | 历史趋势可查询 |

## 三、组件规格

### Gate Progress 组件

**功能**: 显示三个 Gate 的状态和进度

**状态定义**:
| 状态 | 图标 | 说明 |
|------|------|------|
| Completed | ✅ | Gate 已通过 |
| In Progress | 🟡 | 当前 Gate |
| Pending | ⏳ | 未开始 |
| Blocked | 🔴 | 被阻塞 |

**交互**:
- 点击 Gate 卡片查看详情
- 悬停显示 Gate 描述

### Current Gate Status 组件

**字段**:
| 字段 | 说明 |
|------|------|
| Gate | 当前 Gate 编号和名称 |
| Phase | 当前阶段 |
| Day | 观察日进度 |
| Status | 整体状态 (Normal/Warning/Critical) |

### Time Remaining 组件

**显示**:
- 距离 Gate 决策的时间
- 倒计时格式: "X days Y hours"

### Hard Conditions 表格

**列定义**:
| 列 | 说明 |
|----|------|
| Condition | 条件名称 |
| Threshold | 阈值要求 |
| Current | 当前值 |
| Status | 状态 (✅/⏳/❌) |

**阈值颜色**:
- 🟢 满足阈值
- 🟡 接近阈值 (80%)
- 🔴 超出阈值

### Soft Conditions 表格

**同 Hard Conditions，但为建议性条件**

### Risk Alerts 组件

**显示**:
- 当前活跃的风险
- 风险级别 (P0/P1/P2)
- 最后检查时间

**空状态**: "🟢 No active risks"

## 四、数据规格

### 数据来源

| 数据 | 来源 | 更新频率 |
|------|------|---------|
| Gate 状态 | 配置/状态文件 | 手动 |
| 观察天数 | 计算 | 每日 |
| P0/P1 事件 | 事件日志 | 实时 |
| 错误率 | 指标收集 | 5 分钟 |
| P99 延迟 | 指标收集 | 5 分钟 |
| 内存增长 | 指标收集 | 5 分钟 |
| 可用性 | 健康检查 | 1 分钟 |
| 日报状态 | 文件检查 | 每日 |
| 冒烟测试 | 测试执行 | 每日 |

### API 端点

```
GET /api/gate-status
Response:

      "threshold": "≥ 7",

      "status": "pending"
    }
  ],
  "soft_conditions": [...],
  "risks": []
}
```

## 五、交互规格

### 刷新

- **手动刷新**: 点击 [Refresh] 按钮
- **自动刷新**: 每 30 秒
- **刷新动画**: 旋转图标

### 详情查看

- **点击 Gate 卡片**: 展开 Gate 详情面板
- **点击条件行**: 显示条件历史趋势

### 预警通知

- **接近阈值**: 🟡 黄色边框闪烁
- **超出阈值**: 🔴 红色边框 + 通知

## 六、Gray 10% 限制

| 功能 | Gray 10% 状态 | 说明 |
|------|--------------|------|
| 数据展示 | ✅ 启用 | 只读展示 |
| 手动刷新 | ✅ 启用 | 允许刷新 |
| 自动刷新 | ✅ 启用 | 30 秒间隔 |
| Gate 切换 | 🔒 禁用 | 只读，不可操作 |
| 条件修改 | 🔒 禁用 | 只读，不可修改 |
| 阈值调整 | 🔒 禁用 | 只读，不可调整 |

## 七、实现建议

### 技术栈

- **前端**: React + TypeScript
- **状态管理**: Zustand / Redux
- **图表**: Recharts / Chart.js
- **实时**: WebSocket / SSE

### 性能目标

- 首屏加载 < 2s
- 数据更新 < 1s
- 刷新延迟 < 500ms
