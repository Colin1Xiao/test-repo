## 一、放量范围草案

### 阶段 1: 灰度 10% (Week 1)

**目标**: 验证生产环境基础稳定性

**范围**:
- 10% 流量切换到新协调层
- 3 实例生产集群
- 仅读操作 + 低频写操作

**指标**:
- 错误率 < 0.1%
- P99 延迟 < 200ms
- 无 owner 漂移

**退出条件**:
- 连续 7 天无 P0/P1 事件
- 指标达标

---

### 阶段 2: 灰度 50% (Week 2-3)

**目标**: 验证中等负载下稳定性

**范围**:
- 50% 流量切换
- 5 实例生产集群
- 全量操作

**指标**:
- 错误率 < 0.1%
- P99 延迟 < 200ms
- 内存增长 < 10MB/h

**退出条件**:
- 连续 14 天无 P0/P1 事件
- B3 测试通过

---

### 阶段 3: 全量 100% (Week 4)

**目标**: 生产环境全量验证

**范围**:
- 100% 流量切换
- 5 实例生产集群
- 全量操作 + storm 场景

**指标**:
- 错误率 < 0.05%
- P99 延迟 < 150ms
- 资源使用稳定

**退出条件**:
- 连续 7 天无 P0/P1 事件
- 性能指标优于旧系统

## 三、回滚条件草案

### P0: 立即回滚

**触发条件**:
- owner_drift_count > 0
- duplicate_process_count > 0
- 数据丢失/损坏
- 服务不可用 > 5 分钟

**回滚流程**:
1. 自动触发告警
2. 自动切换到旧系统
3. 通知 on-call
4. 事后分析 (24h 内)

---

### P1: 降级 + 调查

**触发条件**:
- 错误率 > 1%
- P99 延迟 > 500ms
- 内存泄漏 > 500MB/24h

**响应流程**:
1. 降低灰度比例 (50% → 10%)
2. 通知 on-call
3. 调查根因 (4h 内)
4. 修复/回滚决策

---

### P2: 观察 + 优化

**触发条件**:
- 性能退化 > 20%
- 资源增长接近阈值
- 偶发错误 (< 0.1%)

**响应流程**:
1. 记录指标
2. 每日审查
3. 排期优化
4. 无需立即行动
