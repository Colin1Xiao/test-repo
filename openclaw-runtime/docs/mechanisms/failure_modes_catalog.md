## 一、分类体系

### 1.1 故障层级

| 层级 | 范围 | 示例 |
|------|------|------|
| L1 | 单点故障 | 文件写入失败、锁获取失败 |
| L2 | 组件故障 | Redis 断连、持久化层损坏 |
| L3 | 系统故障 | 服务崩溃、数据不一致 |
| L4 | 外部依赖故障 | OKX API 不可用、Telegram Bot 异常 |

### 1.2 故障类型

| 类型 | 描述 | 检测方式 |
|------|------|---------|
|  transient | 瞬时故障，可重试 | 重试成功 |
| `permanent` | 永久故障，需修复 | 重试失败 |
| `byzantine` | 拜占庭故障，行为异常 | 一致性检查 |

## 二、持久化层故障

### 2.1 文件写入失败

**故障码**: `PERSIST_WRITE_FAILED`

**症状**:
- JSONL 文件未更新
- 内存索引与文件不一致
- 重启后数据丢失

**原因**:
1. 磁盘空间不足
2. 文件权限问题
3. 并发写入冲突

**检测**:
```bash
# 检查文件大小
ls -lh data/*/incidents.jsonl

# 检查文件权限
ls -l data/*/incidents.jsonl

# 检查磁盘空间
df -h
```

**恢复**:
1. 释放磁盘空间
2. 修复文件权限
3. 从备份恢复

**预防**:
- 文件锁保护写入
- 定期备份
- 磁盘空间监控

### 2.2 JSONL 损坏

**故障码**: `JSONL_CORRUPTED`

**症状**:
- 启动时解析失败
- 部分行无法解析
- 快照与增量不一致

**原因**:
1. 写入中断 (半写入)
2. 并发写入冲突
3. 磁盘坏道

**检测**:
```bash
# 验证 JSONL 有效性
while read line; do echo "$line" | jq . > /dev/null || echo "CORRUPT: $line"; done < incidents.jsonl
```

**恢复**:
1. 删除损坏行
2. 从备份恢复
3. 手动修复 (如可能)

**预防**:
- 原子写入 (写临时文件 + rename)
- 文件锁保护
- 定期校验

### 2.3 快照失效

**故障码**: `SNAPSHOT_INVALID`

**症状**:
- 启动时快照加载失败
- 快照与增量不匹配
- 恢复后状态不一致

**原因**:
1. 快照写入中断
2. 快照版本不兼容
3. 快照文件损坏

**检测**:
```bash
# 验证快照有效性
cat incidents_snapshot.json | jq . > /dev/null

# 检查快照时间戳
cat incidents_snapshot.json | jq '.snapshot_at'
```

**恢复**:
1. 删除快照，从 JSONL 重建
2. 从备份恢复
3. 全量回放 JSONL

**预防**:
- 快照备份 (保留最近 3 个)
- 快照校验和
- 快照后验证

## 三、文件锁故障

### 3.1 锁获取失败

**故障码**: `LOCK_ACQUIRE_FAILED`

**症状**:
- 写入操作超时
- 请求堆积
- 服务响应变慢

**原因**:
1. 锁未释放 (死锁)
2. 锁持有时间过长
3. 并发竞争激烈

**检测**:
```bash
# 检查锁残留
ls -la data/locks/

# 检查锁年龄
find data/locks/ -name "*.lock" -mmin +10
```

**恢复**:
1. 清理陈旧锁
2. 重启服务
3. 分析锁持有原因

**预防**:
- 锁超时自动释放
- 锁年龄监控
- 锁获取失败告警

### 3.2 锁泄漏

**故障码**: `LOCK_LEAK`

**症状**:
- 锁文件残留
- 锁目录增长
- 锁获取失败率上升

**原因**:
1. 异常退出未释放锁
2. 释放逻辑缺陷
3. 陈旧锁清理失效

**检测**:
```bash
# 统计锁文件数
ls data/locks/*.lock | wc -l

# 检查陈旧锁
find data/locks/ -name "*.lock" -mmin +10
```

**恢复**:
1. 清理所有锁文件
2. 重启服务
3. 验证无副作用

**预防**:
- 进程退出时清理锁
- 定期清理陈旧锁
- 锁泄漏告警

### 3.3 锁争用

**故障码**: `LOCK_CONTENTION`

**症状**:
- 锁获取延迟上升
- 写入吞吐量下降
- 超时错误增加

**原因**:
1. 高并发写入
2. 锁粒度过大
3. 锁持有时间过长

**检测**:
```bash
# 监控锁等待时间
# (需要 instrumentation)
```

**恢复**:
1. 降低并发度
2. 优化锁粒度
3. 缩短锁持有时间

**预防**:
- 细粒度锁
- 异步写入
- 批量写入

## 五、重启恢复故障

### 5.1 恢复失败

**故障码**: `RECOVERY_FAILED`

**症状**:
- 服务启动失败
- 数据恢复不完整
- 状态不一致

**原因**:
1. 快照损坏
2. JSONL 损坏
3. 恢复逻辑缺陷

**检测**:
```bash
# 查看启动日志
cat /tmp/server.log | grep -E "Loaded|Recovery|Failed"
```

**恢复**:
1. 从备份恢复
2. 手动修复数据
3. 重建索引

**预防**:
- 备份验证
- 恢复测试
- 渐进式恢复

### 5.2 幽灵状态

**故障码**: `GHOST_STATE`

**症状**:
- 恢复后出现不存在的数据
- 状态与预期不符
- 重复回放

**原因**:
1. 重复回放
2. 快照与增量重叠
3. 索引重建错误

**检测**:
```bash
# 对比恢复前后数据
# (需要基线记录)
```

**恢复**:
1. 重新恢复
2. 手动清理幽灵数据
3. 从备份恢复

**预防**:
- 回放幂等
- 快照边界清晰
- 恢复验证
