# OpenClaw 系统巡检模板

**使用说明**: 复制此模板，按顺序执行各段检查，输出三层报告后归档。

---

## 流程概览

```
原始采集 → 矛盾校正 → 补证遗留项 → 输出三层报告 → 归档
```

---

## 一、系统基础信息

**必查项**:

| 项目 | 命令 | 预期输出 |
|------|------|----------|
| 时间/时区 | `date && date +%Z%z` | UTC+8 |
| CPU 架构 | `uname -m` | x86_64 或 arm64 |
| macOS 版本 | `sw_vers` | 版本号 |
| Darwin 内核 | `uname -a` | 内核版本 |
| OpenClaw 版本 | `openclaw --version` | 版本号+commit |

**输出格式**:
```
时间: YYYY-MM-DD HH:MM UTC+8
主机: [hostname] ([arch])
系统: macOS X.X / Darwin X.X
版本: OpenClaw X.X.X (commit)
```

## 二、核心组件核验

### 2.1 Gateway

| 检查项 | 命令/操作 | 通过标准 |
|--------|----------|----------|
| 进程存在 | `ps aux \| grep openclaw-gateway` | PID 存在 |
| 端口监听 | `lsof -i :18789` 或 `openclaw gateway status` | 127.0.0.1:18789 |
| 健康接口 | `curl -s http://127.0.0.1:18789/health` | `{"ok":true}` |
| 内存占用 | `ps aux` 查看 RSS | 记录数值，观察趋势 |

### 2.2 Telegram

| 检查项 | 命令/操作 | 通过标准 |
|--------|----------|----------|
| 配置存在 | `cat ~/.openclaw/openclaw.json \| grep telegram` | 有配置 |
| Token 有效 | `curl -s "https://api.telegram.org/bot<TOKEN>/getMe"` | `{"ok":true}` |

### 2.3 Memory Search

| 检查项 | 命令/操作 | 通过标准 |
|--------|----------|----------|
| 配置启用 | 检查 `memorySearch.enabled` | true |
| 模型文件 | `ls ~/.openclaw/memory/models/` | 文件存在 |
| **实际检索** | `openclaw memory search "test" --json` | 返回结果 |
| 索引状态 | `sqlite3 ~/.openclaw/memory/main.sqlite "SELECT COUNT(*) FROM files;"` | 文件数 |

### 2.4 调度任务

| 检查项 | 命令 | 说明 |
|--------|------|------|
| crontab | `crontab -l` | 通常为空 |
| launchd | `ls ~/Library/LaunchAgents/ \| grep -v Apple` | OpenClaw 相关任务 |

## 三、资产统计

### 3.1 磁盘

```bash
df -h / /System/Volumes/Data
```

**注意**: APFS 容器共享空间，可用值不能相加。

### 3.2 技能与脚本

```bash
# 技能目录数
ls -d ~/.openclaw/workspace/skills/*/ 2>/dev/null | wc -l

# 脚本文件数
ls ~/.openclaw/workspace/scripts/*.sh 2>/dev/null | wc -l
ls ~/.openclaw/workspace/scripts/*.py 2>/dev/null | wc -l
```

### 3.3 记忆与日志

```bash
# memory 文件数
ls ~/.openclaw/workspace/memory/*.md 2>/dev/null | wc -l

# logs 文件数
ls ~/.openclaw/workspace/logs/ | wc -l

# sessions 条目数
ls ~/.openclaw/workspace/sessions/ | wc -l
```

## 五、问题与修正

**必查项**:

1. 数量是否与上次一致？（漂移检测）
2. 状态词是否过强？（"就绪"→"已验证可用"）
3. 数字是否自洽？（百分比可回推）
4. 是否有新增错误日志？

**日志扫描**:
```bash
tail -100 /tmp/openclaw/openclaw-$(date +%Y-%m-%d).log | grep -iE "(error|fail|panic|timeout)"
```

## 六、输出物

### 文件命名规范

| 类型 | 文件名 | 用途 |
|------|--------|------|
| 事实底稿 | `system-asset-report-YYYYMMDD.md` | 完整详细报告 |
| 日常查看版 | `system-summary-YYYYMMDD.md` | 简版摘要 |
| 管理层版 | `openclaw-executive-summary-YYYY-MM-DD.md` | 一页汇报 |

### 归档位置

```
reports/
├── system-asset-report-YYYYMMDD.md
├── system-summary-YYYYMMDD.md
├── openclaw-executive-summary-YYYY-MM-DD.md
└── .inspection-template.md  # 本模板
```

## 状态标注规范

| 标注 | 含义 |
|------|------|
| ✅ 已确认 | 有直接命令输出/接口返回支持 |
| ⚠️ 待验证 | 证据不完整，方向明确 |
| ❌ 存疑 | 无法证明，需进一步调查 |
| 💡 建议 | 非问题，但有优化空间 |

## 快速执行脚本

```bash
# 一键采集核心信息
echo "=== $(date) ===" && \
echo "架构: $(uname -m)" && \
echo "版本: $(openclaw --version 2>/dev/null | head -1)" && \
echo "Gateway: $(curl -s http://127.0.0.1:18789/health 2>/dev/null)" && \
echo "Memory: $(openclaw memory search 'test' --json --max-results 1 2>/dev/null | grep -c 'results')" && \
echo "技能: $(ls -d ~/.openclaw/workspace/skills/*/ 2>/dev/null | wc -l | tr -d ' ')" && \
echo "磁盘: $(df -h / | tail -1 | awk '{print $5}')"
```
